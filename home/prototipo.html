CONTEÚDO DO MEU TRABALHO Projeto de Banco de Dados Sistema de Vendas de Jogos – Loja Digital Aluno: Fernando do Carmo Stofeles Faculdade: Uniasselvi Curso: Análise e Desenvolvimento de Sistemas Disciplina: Imersão Profissional: Projeto de Banco de Dados (173474) 1. Introdução O mercado de jogos digitais tem crescido exponencialmente devido à evolução tecnológica e à popularização dos jogos em diversas plataformas. Este projeto visa criar um banco de dados para uma loja digital de venda de jogos, facilitando o gerenciamento das informações de clientes, produtos, pedidos e pagamentos, garantindo eficiência e organização. 2. Objetivo Desenvolver um banco de dados relacional para uma loja digital de jogos, capaz de armazenar, consultar e manipular dados relativos a clientes, jogos, pedidos, pagamentos e entregas, com segurança e desempenho. 3. Metodologia O desenvolvimento seguiu as etapas: levantamento de requisitos, modelagem conceitual (DER), modelagem lógica, implementação do modelo físico com comandos SQL, e criação de consultas básicas para demonstração. Para o DER foi usado o Draw.io, e para o banco de dados, MySQL. 4. Modelagem Conceitual – Diagrama Entidade-Relacionamento (DER) Entidades principais: Cliente Atributos: id_cliente (PK), nome, email, telefone, data_cadastro Jogo Atributos: id_jogo (PK), nome, descricao, preco, genero, data_lancamento Pedido Atributos: id_pedido (PK), id_cliente (FK), data_pedido, status Item_Pedido Atributos: id_item (PK), id_pedido (FK), id_jogo (FK), quantidade, preco_unitario Pagamento Atributos: id_pagamento (PK), id_pedido (FK), tipo_pagamento, data_pagamento, valor_pago Relacionamentos: Um Cliente pode fazer vários Pedidos (1:N) Um Pedido contém vários Itens (1:N) Cada Item_Pedido refere-se a um Jogo (N:1) Cada Pedido tem um Pagamento (1:1) 5. Modelo Físico – Código SQL (Exemplos) CREATE TABLE Cliente ( id_cliente INT AUTO_INCREMENT PRIMARY KEY, nome VARCHAR(100) NOT NULL, email VARCHAR(100) NOT NULL UNIQUE, telefone VARCHAR(20), data_cadastro DATE NOT NULL ); CREATE TABLE Jogo ( id_jogo INT AUTO_INCREMENT PRIMARY KEY, nome VARCHAR(100) NOT NULL, descricao TEXT, preco DECIMAL(10,2) NOT NULL, genero VARCHAR(50), data_lancamento DATE ); CREATE TABLE Pedido ( id_pedido INT AUTO_INCREMENT PRIMARY KEY, id_cliente INT NOT NULL, data_pedido DATETIME NOT NULL, status VARCHAR(20) NOT NULL, FOREIGN KEY (id_cliente) REFERENCES Cliente(id_cliente) ); CREATE TABLE Item_Pedido ( id_item INT AUTO_INCREMENT PRIMARY KEY, id_pedido INT NOT NULL, id_jogo INT NOT NULL, quantidade INT NOT NULL, preco_unitario DECIMAL(10,2) NOT NULL, FOREIGN KEY (id_pedido) REFERENCES Pedido(id_pedido), FOREIGN KEY (id_jogo) REFERENCES Jogo(id_jogo) ); CREATE TABLE Pagamento ( id_pagamento INT AUTO_INCREMENT PRIMARY KEY, id_pedido INT NOT NULL, tipo_pagamento VARCHAR(50), data_pagamento DATETIME, valor_pago DECIMAL(10,2), FOREIGN KEY (id_pedido) REFERENCES Pedido(id_pedido) ); 7. Consultas SQL de exemplo Listar todos os jogos disponíveis: sql Copiar Editar SELECT id_jogo, nome, preco, genero FROM Jogo; Consultar pedidos de um cliente: sql Copiar Editar SELECT p.id_pedido, p.data_pedido, p.status FROM Pedido p JOIN Cliente c ON p.id_cliente = c.id_cliente WHERE c.email = 'cliente@exemplo.com'; Inserir um novo cliente: sql Copiar Editar INSERT INTO Cliente (nome, email, telefone, data_cadastro) VALUES ('Fernando Stofeles', 'fernando@example.com', '21999999999', CURDATE()); Atualizar status do pedido: sql Copiar Editar UPDATE Pedido SET status = 'Pago' WHERE id_pedido = 1; 8. Considerações sobre a Implementação Neste projeto, o foco principal foi a modelagem e implementação do banco de dados para a loja digital de vendas de jogos. O código SQL que define as tabelas, relacionamentos e restrições foi desenvolvido para criar a estrutura do banco de dados. Não foi desenvolvida uma interface gráfica (frontend) para este sistema, pois o objetivo da disciplina é aprofundar os conhecimentos em modelagem e manipulação de dados utilizando SQL. Portanto, o código SQL apresentado neste relatório está separado da interface visual e serve para demonstrar a criação e organização do banco de dados, além de possibilitar a execução das consultas e operações necessárias para o funcionamento do sistema. 10.
